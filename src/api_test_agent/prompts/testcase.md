# 输出格式要求

请严格按照以下 Markdown 格式输出测试用例：

## {METHOD} {PATH}

> {SUMMARY}

| 编号 | 场景 | 输入 | 预期状态码 | 预期响应 | 优先级 |
|------|------|------|-----------|---------|--------|
| TC-XXX | 场景描述 | 请求参数/请求体 JSON | 状态码 | 预期响应描述 | P0/P1/P2 |

## 规则
- 编号全局递增，格式 TC-001, TC-002, ...
- 输入列：对于 GET 请求写 query 参数，对于 POST/PUT 写请求体 JSON
- 优先级：P0=核心正常流程, P1=重要异常场景, P2=边缘场景
- 每个接口的第一个用例必须是正常场景（P0）
